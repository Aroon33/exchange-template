# CryptoX Exchange UI Template

このリポジトリは、仮想通貨取引所のフロントエンド UI を確認するための静的デモです。`index.html` や `exchange.html` をはじめ、ウォレット、システム取引、KYC、履歴確認など、実際のプロダクトで必要となる主要な画面が用意されています。`assets/` 以下に共通のスタイルとアイコンがまとまっており、API 連携前でも画面全体の動きをデモできます。

## できること / お手伝いできること
- **画面カスタマイズ**: 色・タイポグラフィ・コンポーネントの微調整、共通ヘッダー/フッターのナビゲーション整理、LP コピー改善など。
- **機能モックの追加**: 注文板やウォレット残高にモックデータを流す、ステップ表示やバリデーション文言を整えるなど、UX を把握しやすくするための改善。
- **API 連携の下地づくり**: 既存ボタンやフォームに `data-*` 属性やクラスを付与し、実装担当者がイベントハンドラを取り付けやすい形にする。
- **アクセシビリティ / 多言語対応の補助**: aria 属性やフォーカス順の整理、文言の i18n 化に備えたリソース分離など。
- **動線設計の提案**: 口座開設〜入金〜取引までの導線を短くするための CTA 配置、導入ツアーやヘルプリンクの挿入案。

## 主要ファイル
- `index.html`: LP 兼トップページ。ヒーロー、特徴、ステップ、FAQ などを含みます。
- `exchange.html`: 板・約定履歴・注文パネルをまとめたトレード画面（デモ）。
- `wallet.html`: 残高、入出金フォーム、履歴などウォレット関連 UI。
- `system-trade.html`: 戦略テンプレート、ロット設定、リスクコントロールの入力 UI。
- `assets/css/style.css`: 全体のスタイルとレイアウト。
- `js/`: 追加のインタラクションが必要な場合に利用するためのスクリプト置き場（現状は空に近い構造）。

## 次のステップ例
1. **デザイン微調整**: ブランドカラーやロゴを入れ替えて、最終デザインに近づける。
2. **モックデータ挿入**: `exchange.html` の板や `wallet.html` の残高に実データに近いサンプルを入れ、デモ精度を上げる。
3. **イベントフック追加**: ボタンやフォームに ID/クラスを付与し、バックエンド/API 実装時にそのままフックできるようにする。
4. **コンテンツ整備**: FAQ、キャンペーン、サポート文言を運用に即した内容へ差し替える。

具体的な改修内容やご要望があれば、個別のファイルや画面単位で対応案を提示・実装できます。お気軽にお知らせください。
